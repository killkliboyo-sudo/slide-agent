# 自動化簡報生成系統：設計與開發計畫

## 背景與目標

製作高品質的簡報需要耗費大量時間和精力，從理解原始資料、摘取重點，到設計視覺元素都需要專業知識。對研究人員而言，將學術論文濃縮為簡報尤為困難[1]。雖然大型語言模型（LLM）等生成式技術已廣泛應用於內容產生，但單純依賴這些模型往往難以處理多樣的資料形式並確保內容的真實與精確[1]。

使用者希望將現有的「助手」升級為一個智慧代理（Agent），自動完成從資料分析到簡報輸出的整個流程，以節省人力並提高品質。

### 目標

開發一個自動化簡報生成代理系統，能夠接受多種來源的資料（數據表格、技術報告、學術論文、文本檔案等），以及使用者提供的提示詞或特定需求，經過分析後直接產出結構完善的幻燈片簡報檔案。

為達成此目標，系統需完成以下階段性任務：

1. **解析與理解輸入資料**：讀取並解析使用者提供的各類檔案，透過自然語言處理提取關鍵概念、重要數據和論證關係，理解使用者的附加說明或意圖，形成結構化的內容摘要。

2. **制定簡報大綱**：根據摘要內容和使用者需求，預估所需的投影片數量（可參考「一分鐘一張投影片」原則[2]），列出每頁投影片的主題與標題。此步驟遵循「單一概念法則」，確保每張投影片只傳達一個重點[3]；並採用「標題即結論」原則撰寫標題，即將該頁的結論性訊息直接體現在標題上[4]。

3. **逐頁內容撰寫與版面設計**：依據大綱為每張投影片撰寫精簡的文字說明，包含必要的數據和解釋。同時運用文字生成圖像模型，產生與主題相符的抽象背景圖或插圖（不含文字的純圖形），預留空間以放置文字方塊和圖表。為每頁設計版面配置，包括標題位置、文字區塊與圖片區塊，以統一定義的風格（如配色、字體、頁面佈局）呈現內容。

4. **輸出完整簡報**：最後，將各頁的文字內容、生成的背景圖片和可能的圖表組合起來，產出一份完整的簡報檔案（例如 PowerPoint PPTX 檔）。簡報的視覺風格在整份文件中保持一致，採用16:9比例，高對比配色和無襯線易讀字體，並在適當位置附上資料來源註記。完成的簡報將提供給使用者下載或瀏覽。

透過上述步驟，本系統代理可以自動消化大量原始資料，並快速生成符合使用者需求的高品質簡報，減少人工製作簡報的負擔。

## 簡報設計指引

為確保生成的簡報清晰易懂且具備專業水準，我們將在簡報內容與設計上遵循公認的原則。以下彙總幾項學術界與業界推薦的簡報設計準則，每項均附上出處：

### 核心設計原則

- **單一概念法則**：每張投影片只聚焦一個核心訊息；若內容過於複雜，寧可拆分成多張投影片逐一呈現[3]。這樣可避免一張投影片承載過多資訊而使重點模糊不清。

- **1 分鐘規則**：建議講解每張投影片的時間約為 1 分鐘，藉此推算投影片頁數並控制每頁內容長度[2]。如果某頁內容講解需時遠超過1分鐘，表示該頁資訊過載，應拆成多頁或刪減次要細節[2]。

- **標題即結論**：投影片的標題直接傳達本頁的結論或關鍵訊息，而非籠統的分類（例如避免只寫「結果」或「結論」作為標題）[4]。觀眾即使只看標題，也能瞭解此頁要點。

- **保留必要資訊**：投影片上只放置演講中會提及的內容；不會口頭講述的細節可移除或放入附錄頁面[5]。精簡內容可降低觀眾的認知負荷，同時避免次要資訊分散注意力。

- **善用圖像輔助**：除非必要，應避免投影片上出現大段文字說明。盡量以圖表或圖片來呈現資訊[6]（「一張圖勝過千言萬語」），並確保每張投影片至少包含一個視覺元素。然而也要避免在單頁放入過多或過於複雜的圖片/圖表，以免主旨不清[6]。

- **控制元素數量**：每頁投影片上的元素（文字塊、圖形、圖表等）不宜超過 6 個，以免造成認知過載[7]。研究指出，人腦易於解讀約6個資訊元件，再多則認知負荷將急劇增加500%[8]。因此簡報設計應嚴格篩選重點，寧缺毋濫。

- **視覺階層與一致性**：專業簡報強調內容的視覺階層、對比、留白，以及字體與圖形風格的一致性[9]。透過合理運用字體大小與粗細來建立層次關係，搭配高對比度的色彩和留白區域引導視線，可以大幅提升資訊傳達的效率。同時，全篇簡報應維持統一的設計風格（配色方案、字型、圖表樣式等），以展現專業性[9]。

上述原則將作為本系統在生成簡報時的指導方針。例如，代理在預估簡報頁數時會應用「1分鐘規則」來確定適當的頁數，不讓單一投影片承載過多內容；在版面配置時嚴格限制每頁元素數量並選用易讀的無襯線字體和高對比配色。這確保即便觀眾走神，看到投影片的一瞬間也能抓取關鍵重點[10]。

## 系統架構與工作流程

為了實現上述功能，我們採用多模組子代理（Agent）協作的系統架構。這種架構類似由多位專家組成的團隊，各司其職又相互配合，逐步將原始資料轉化為完整簡報。近期的研究也證實，多代理協作是提升自動簡報生成效果的有效方法[11]。

### 系統架構圖

```
[使用者輸入] → [資料分析子代理] → [大綱生成子代理] → [內容與設計子代理] → [簡報組裝子代理] → [輸出簡報檔案]
```

基於以上理念，本提案將系統劃分為四個主要子代理模組，按照順序執行各自的任務並傳遞結果：

### 1. 資料分析子代理（Data Analysis Agent）

**輸入**：使用者提供的各類資料檔案（例如文字報告、PDF 論文、試算表、數據集）以及使用者的提示詞或特定要求。

**處理**：透過自然語言處理（NLP）與資料解析技術來閱讀並理解輸入內容。此代理將抽取文件中的關鍵概念、重要數字和圖表資訊，以及各段落的重點論述。對於結構化資料（如表格），則分析其中的趨勢或異常值。若使用者提供了提示詞（例如希望強調某主題或觀點），代理會據此強化相關內容的提取。必要時，可運用預訓練的大型語言模型來協助摘要和資訊抽取，但同時會搭配規則或額外檢核以確保資訊準確無誤。

**輸出**：經過結構化的內容摘要，包含主要章節或主題、各章節下的重要發現或論點，以及支撐這些論點的依據（數據、引文等）。這份摘要將作為後續大綱生成的基礎。

> **註**：此步驟對應於學術研究中的內容理解與結構化階段，例如 Auto-Slides 系統中的 Parser 和 Planner 代理合作完成初步解析與簡報計畫[11]。

### 2. 大綱生成子代理（Outline Generator）

**輸入**：資料分析代理產出的重點摘要，以及使用者的額外需求或偏好（例如簡報預期風格、受眾類型、希望突出的部分等）。

**處理**：根據內容摘要來制定整份簡報的框架結構。首先，估算所需的投影片頁數：參考「1分鐘規則」，如果預期簡報講解時間為 N 分鐘，則大致規劃 N 張投影片[2]（可視內容複雜度調整）。接著，按照「單一概念法則」將摘要內容劃分到各投影片，確保每頁聚焦單一主題[3]。為每張投影片撰寫標題，直接點明該頁的結論或核心訊息（應對應摘要中的一個重點）[4]。同時列出每頁需要呈現的要點清單（通常3–5點簡述）以及支援這些要點的資料（數據指標、引文等）。

如果某些要點適合圖表或圖片展示，註明圖表類型或圖片主題（例如「柱狀圖比較銷量趨勢」或「產品架構示意圖」）。另外，根據使用者需求及內容特性，為簡報選定視覺風格方向，例如配色主題（嚴肅商業風、科技藍色系等）、版面模板（16:9比例，橫向排版或直式排版）等，作為後續設計的指引。

**輸出**：簡報大綱，包括每頁的標題、重點要點清單、預計使用的圖表/圖片種類、以及全局的設計風格說明。大綱既是內容藍圖，也是設計藍圖，確保下一步創作時有清晰的方向。

> 此階段相當於人工製作簡報時的構思與策劃，也類似於研究中的「Planner Agent」為每張投影片產生結構化計畫[11]。

### 3. 內容與設計子代理（Slide Designer）

**輸入**：完整的簡報大綱，包括每頁的主題標題、要點清單、圖表/圖片需求以及設計風格規範。

**處理**：按照大綱逐頁生成投影片內容和設計。對於每一頁：

#### 文字內容撰寫
根據該頁的大綱要點撰寫簡潔的敘述文字。文字應開門見山地說明該頁主旨，並輔以必要的背景資訊或數據說明。考慮到投影片上的文字不宜過多，每個要點通常以一句話概括，整頁加起來不超過5-6行箇條式文字，並與講者口頭講述相配合[12][13]。代理可運用語言模型將摘要內容改寫為適合投影片呈現的短句，確保文字精煉且易懂。

#### 圖像內容創作
依照大綱需求，為每頁生成對應的圖表或圖片佈景。如果大綱指定了圖表類型且給出了數據，則可利用繪圖函式庫自動繪製圖表（例如 matplotlib 繪製折線圖）。若需插入圖片（例如示意圖或裝飾性背景），則調用文字生成圖像模型（例如本系統內建的 imagegen 工具）根據頁面主題創建圖片。在生成圖片時，遵循不直接在圖片中嵌入文字的原則，以便後續文字內容能獲得最佳可讀性。圖片風格會統一遵循大綱指定的主題，例如色調和抽象程度。生成的圖像將儲存供稍後組裝使用。

#### 版面與樣式設計
根據全局風格規範，為該頁確定版面配置（Layout）。包括：標題位置（通常頁頂明顯處）、主文字區的位置（左側、右側或下方等）、以及圖像或圖表區的位置。遵循視覺設計最佳實踐來佈局：突出標題以建立層次（例如標題字體比正文大且醒目）[9]；為文本區域和圖像區域留出足夠留白以避免雜亂[9]；整體配色與字體樣式與全局其他頁保持一致[9]。同時檢查每頁元素數量不超過預期，避免因資訊過多破壞版面平衡[8]。

字體方面，優先使用易讀的無襯線字體，如 Segoe UI 或 Inter，字號充足大以確保投影時清晰可辨。字體樣式（粗體、斜體等）僅用於強調重點，避免使用底線或全大寫等降低可讀性的做法。

#### 資料來源標註
對於每頁的關鍵數據或引述內容，記錄其來源（例如論文來源、數據出處等），準備在投影片頁面上加入小型註腳或引用文字。這對提升簡報可信度很重要[14]。代理會暫存這些來源資訊（如網址或文獻標號），以便在組裝階段統一添加到頁腳或以懸掛文字形式標示在對應內容旁。

**輸出**：每張投影片的素材與排版方案，例如：「第3頁：標題『研究結果 – XXX方法提升了准确度』，背景圖片ID=img_3.png，左側佈局文字區含三條要點，右側佈局放置圖表ID=chart_3.png，字體Segoe UI，全白底+深藍文字，字體大小標題40pt/內文24pt，頁腳加註引用[1]」。

如上，內容與設計子代理會為每頁產生組裝說明，包括該頁的文字內容、所需圖片/圖表的檔案路徑、版面配置細節，以及需要附上的引用註記等。這些說明將提供給下一模組進行最終組裝。

> **值得注意的是**，內容與設計子代理在此階段充分應用了先前定義的簡報設計原則：例如確保每頁都有圖像而不只是文字[6]、限制資訊點的數量以避免觀眾認知過載[7]、以及透過一致的字體和風格保持專業觀感[9]。

### 4. 簡報組裝子代理（Presentation Assembler）

**輸入**：所有投影片頁面的內容與設計說明，包括文字、圖片/圖表檔案，以及版面配置與樣式規範。

**處理**：利用程式介面將前序步驟的成果組合成實際的簡報檔案。具體而言，本模組會採用簡報製作程式庫（例如 python-pptx）或其它Office自動化工具，按頁順序創建投影片物件並填入內容：

#### 設定簡報母版版式
頁面比例16:9，背景底色或樣式符合設計規範，高對比度色彩主題以確保投影清晰度。全域設定無襯線字體（如 Segoe UI，在 Windows 系統上具有卓越的可讀性[15]；或跨平台字體如 Arial），作為預設字型應用於所有文字方塊[15]。也可嵌入先前指定的自訂字體（如 Inter）確保風格一致。

#### 逐頁生成內容
對於每張投影片，插入對應的背景圖片或設定背景色。按照設計子代理提供的佈局資訊，在正確的位置創建標題文本框、內容文本框、圖表或圖片佔位元等元素。將該頁的文字填入文本框，套用預定的字體大小和顏色樣式。將圖表或圖片文件插入並縮放到指定區域。調整各元素的層級關係（例如背景圖置底，文字框置頂）以符合視覺階層要求。

#### 套用一致的樣式
確保所有投影片的轉場、頁碼風格等全域元素統一。使用模板功能將重複的版式套用於多頁，保持簡報切換時風格不跳脫。檢查整份簡報的外觀與預期設計一致，包含色彩是否統一、字體是否正確，以及頁面佈局是否與規劃相符。

#### 添加資料來源與附註
根據設計子代理提供的資料來源資訊，在相應的投影片添加引用標註。例如在頁腳加入小字註明數據來源，或在某個數據點旁以括號標出引用編號。引用格式保持全篇一致，讓觀眾容易分辨出哪些資訊有對應來源[14]。同時建立一頁「參考資料」或於附錄列出完整的引用清單（若適用）。

**輸出**：完整的簡報檔案（如 PPTX 或 PDF）。此檔案包含排版妥當的幻燈片，內嵌所有文字和圖像，並可直接用於演示或進一步編輯。生成過程中若發現任何缺漏（例如某頁缺少對應圖片或文字過長導致排版問題），組裝代理可回饋此資訊給相關子代理進行調整，確保輸出品質。

> 在自動簡報研究中，這部分類似於最終的生成階段[11]；我們也預留了與前序模組互動調整的機制，以實現內容品質保證。

透過上述多個子代理的分工合作，系統能模擬人工團隊製作簡報的流程：從內容擬定、設計創作到最後製作，每一步都有專門模組把關。其中若有任何步驟未達要求，仍可在人為介入下針對該子模組進行微調後重新執行。整體架構設計強調模組化、並行開發與漸進增強，未來也可針對某一模組替換更先進的模型或方法，而不影響其他部分運作。

## 實作示例

為驗證系統構想的可行性，我們使用上述架構實作了一個簡化的範例，產生了兩頁幻燈片的簡報（附檔 demo_presentation.pptx）。該範例展示了代理自動生成簡報的形式與設計風格：

### 投影片 1：研究簡報設計基本原則
介紹研究簡報設計的基本原則，包括：
- 「每頁只傳達一個重點」[3]
- 「每張投影片講解控制在約1分鐘內」[16]
- 「以結論作為頁面標題」[17]等理念

代理將這些要點濃縮為該頁的三條簡潔 Bullet，要點旁註明了出處，以突顯其有據可依。

### 投影片 2：視覺設計建議
提供具體的視覺設計建議，例如：
- 使用高對比度配色並將每頁元素控制在不超過 6 個[7]
- 善用圖片避免出現滿版文字[6]
- 選用易讀的字體並維持良好的視覺階層結構[9]等

這些提示同樣以要點形式列出並附上參考來源。

在該示例中，每張投影片的背景都由文字生成圖像模型自動產生，風格為抽象的淡色調圖形，沒有包含任何文字。為確保文字內容的可讀性，系統在背景圖上加覆了一層半透明的深色遮罩，讓白色的標題和正文文字更加醒目。每個要點旁還標註了資訊來源，方便觀眾在會後查閱詳細資料[14]。最終輸出的 PPT 檔每頁風格一致、美觀專業，充分證明了本代理的整合效能。

## 結論

本計畫提出的自動化簡報生成代理，結合了內容分析與版面設計的能力，能在短時間內將繁雜的原始資料轉化為條理清晰、圖文並茂的簡報成品。透過多子代理的架構設計，系統確保每個環節都經過專門優化：從重點資訊的精確提取，到符合設計原則的幻燈片創作，都有相應模組把關，避免了單一模型生成易出錯、難控品質的缺點[1]。使用者僅需提供資料和希望強調的重點，即可自動獲得高品質的簡報初稿，大幅降低製作門檻。

值得強調的是，系統在簡報設計上內建了學術與業界驗證的最佳實踐作為指導，產出的幻燈片不僅內容正確，還具有專業水準的視覺表現力。此代理可廣泛應用於商業匯報、學術講座、教育培訓等場合，為使用者節省時間的同時提高簡報效果。

未來，隨著技術進步與更多設計範例的累積，我們可以持續擴充代理的模板庫與風格知識，使其能生成更豐富多樣、貼合特定領域需求的簡報。此外，我們也考慮引入互動式優化機制，允許使用者在生成後透過自然語言指令對簡報進行細調，代理會即時更新內容並產生新的版本[18]。這將進一步提升系統的實用性和靈活性。

綜上所述，本自動化簡報生成系統藉由先進的NLP和生成技術，配合嚴謹的設計原則與模組化架構，實現了從資料到簡報的高效轉換。它有望成為各領域人士節省時間、提升溝通效果的強大工具。藉由不斷的優化迭代，該系統將朝著產出更高品質、自訂性更強的簡報方向持續演進。
